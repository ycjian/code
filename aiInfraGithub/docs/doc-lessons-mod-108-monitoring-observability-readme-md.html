Module 08: 监控（Monitoring） & 可观测性（Observability）

概览（Overview）

This module covers 全面监控（comprehensive monitoring） and observability practices for AI infrastructure. You'll learn to implement production-grade monitoring systems that track infrastructure health, application performance, and ML model behavior in real-time.

Module Objectives

By the end of this module, you will be able to:

  1. Understand 可观测性（Observability）: Explain the three pillars of observability (metrics, logs, traces)
  2. Implement Metrics Collection: Set up Prometheus 指标监控（Prometheus） for infrastructure and application metrics
  3. Create Dashboards: Build Grafana 可视化（Grafana） dashboards for visualizing system health
  4. Centralize Logging: Deploy ELK/EFK stack for log aggregation and analysis
  5. Distributed Tracing: Implement request tracing across microservices
  6. Set Up 告警（Alerting）: Configure intelligent alerts and on-call workflows
  7. Monitor ML Models: Track model performance, drift, and data quality
  8. 生产环境（Production） Operations: Implement SLIs, SLOs, and incident response

前置要求（Prerequisites）

  • Required:
  • Linux command line proficiency
  • Docker 容器（Docker） fundamentals (Module 03)
  • Kubernetes 容器编排（Kubernetes） basics (Module 04)
  • Basic networking concepts

  • Recommended:

  • Understanding of HTTP/REST APIs
  • Basic SQL knowledge

Key Concepts Covered

可观测性（Observability） Fundamentals
  • Metrics: Numerical measurements over time (CPU, memory, requests/sec)
  • Logs: Discrete events with context (errors, transactions)
  • Traces: Request flow across 分布式系统（distributed systems）
  • Cardinality: Number of unique metric label combinations
  • Sampling: Reducing data volume while maintaining insights
Prometheus 指标监控（Prometheus） Concepts
  • Time Series: Uniquely identified by metric name + labels
  • Scraping: Pull-based metric collection
  • Exporters: Applications exposing metrics endpoints
  • Federation: Hierarchical Prometheus 指标监控（Prometheus） setup
  • Remote Storage: Long-term metric storage
Grafana 可视化（Grafana） Concepts
  • Data Sources: Where Grafana 可视化（Grafana） queries data from
  • Panels: Individual visualizations
  • Variables: Dynamic dashboard parameters
  • Annotations: Event markers on graphs
  • Playlists: Rotating dashboard displays
Logging Concepts
  • Structured Logging: JSON-formatted log entries
  • Log Levels: DEBUG, INFO, WARN, ERROR, FATAL
  • Correlation IDs: Link related log entries
  • Log Retention: How long to keep logs
  • Index Patterns: Elasticsearch index organization
Tracing Concepts
  • Spans: Individual operations within a trace
  • Trace Context: Propagated across service boundaries
  • Sampling: Percentage of traces to collect
  • Service Mesh: Automatic trace instrumentation
  • Tail-based Sampling: Sample after seeing full trace

Industry Context

Why This Matters

可观测性（Observability） is critical for ML infrastructure:

  • Complexity: Modern ML systems span dozens of services
  • Scale: Processing millions of predictions per day
  • 可靠性（Reliability）: ML systems must meet SLAs (99.9%+ uptime)
  • Cost: GPU 加速（GPU）/compute costs require careful monitoring
  • Compliance: Regulations require audit trails
  • Performance: Sub-second latency requirements
Common Use Cases
  1. Model Performance 监控（Monitoring）: Track accuracy, latency, throughput
  2. Data Quality: Detect drift, missing features, outliers
  3. Infrastructure Health: GPU 加速（GPU） utilization, memory, disk I/O
  4. Cost Attribution: Track costs per model, team, project
  5. 事故响应（Incident Response）: Root cause analysis, debugging
  6. Compliance: Audit logs, data access tracking
