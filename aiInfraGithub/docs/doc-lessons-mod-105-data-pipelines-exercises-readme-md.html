Module 05: Data Pipelines - Hands-On Exercises

概览（Overview）

This directory contains hands-on exercises and labs for Module 05. Complete these exercises to solidify your understanding of data pipelines, Spark 分布式计算（Spark）, Kafka 流处理（Kafka）, and data quality.

Lab 01: Apache Airflow 工作流编排（Airflow） Basics

Duration: 2-3 hours Difficulty: Beginner

目标（Objectives）
  • Set up Apache Airflow 工作流编排（Airflow） locally
  • Create your first DAG
  • Implement task dependencies
  • Use XComs for data passing
  • Schedule and trigger DAGs
Setup
bashBASH
cd lab-01-airflow-basics
./setup.sh
Tasks
  1. Task 1: Create a simple ETL DAG with 3 tasks
  • Extract data from CSV
  • Transform (filter and aggregate)
  • Load to SQLite database
  1. Task 2: Add error handling and retries
  • Configure retry strategy
  • Add failure callbacks
  • Implement alerting
  1. Task 3: Create a branching DAG
  • Use BranchPythonOperator
  • Implement conditional logic
  • Join branches at the end

Solution: solutions/lab-01/

Lab 03: Kafka 流处理（Kafka） Streaming Pipeline

Duration: 3-4 hours Difficulty: Intermediate

目标（Objectives）
  • Set up Kafka 流处理（Kafka） locally
  • Implement producers and consumers
  • Build real-time feature engineering
  • Handle failures with DLQ
Setup
bashBASH
cd lab-03-kafka-streaming
docker-compose up -d
Tasks
  1. Task 1: Simple producer/consumer
  • Create topic user-events
  • Implement event producer
  • Implement basic consumer
  1. Task 2: Real-time feature engineering
  • Consume events from topic
  • Compute windowed aggregations (1h, 24h)
  • Emit features to output topic
  1. Task 3: Error handling
  • Implement retry logic
  • Create dead letter queue
  • Add monitoring metrics
  1. Task 4: Consumer group scaling
  • Run multiple consumer instances
  • Test load balancing
  • Monitor consumer lag

Architecture:

textTEXT
Producer → [user-events] → Feature Engineer → [features] → ML Model
                                    ↓
                            [dead-letter-queue]

Lab 05: End-to-End ML Pipeline

Duration: 6-8 hours Difficulty: Advanced

目标（Objectives）

Build a complete production-grade ML data pipeline that demonstrates all concepts from Module 05.

要求（Requirements）
  • Orchestration: Airflow 工作流编排（Airflow） DAG
  • Data Processing: Spark 分布式计算（Spark） for feature engineering
  • Streaming: Kafka 流处理（Kafka） for real-time inference
  • Quality: Great Expectations validation
  • 监控（Monitoring）: Prometheus 指标监控（Prometheus） metrics
  • Versioning: DVC 数据版本控制（DVC） for data tracking
架构（Architecture）
textTEXT
┌─────────────────────────────────────────────────────────┐
│                     Airflow DAG                         │
│                                                          │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐           │
│  │ Extract  │ → │Transform │ → │Validate  │           │
│  │  (API)   │   │ (Spark)  │   │   (GE)   │           │
│  └──────────┘   └──────────┘   └──────────┘           │
│                        ↓                                 │
│                  ┌──────────┐   ┌──────────┐           │
│                  │  Train   │ → │ Register │           │
│                  │  Model   │   │  Model   │           │
│                  └──────────┘   └──────────┘           │
└─────────────────────────────────────────────────────────┘
                          ↓
              ┌───────────────────────┐
              │   Kafka Streaming     │
              │                       │
              │  Events → Features →  │
              │  Model → Predictions  │
              └───────────────────────┘
Phase 1: Batch 训练（Training） Pipeline (3-4 hours)

Tasks:

  1. Set up project structure
  2. Create Airflow 工作流编排（Airflow） DAG for training
  3. Implement Spark 分布式计算（Spark） feature engineering
  4. Add data quality validation
  5. Train and register model
  6. Version data with DVC 数据版本控制（DVC）

DAG Structure:

pythonPYTHON
extract_data >> validate_raw_data >> transform_features >>
validate_features >> train_model >> register_model
Phase 2: Streaming 推理（Inference） Pipeline (2-3 hours)

Tasks:

  1. Set up Kafka 流处理（Kafka） cluster
  2. Implement event producer
  3. Build feature engineering consumer
  4. Add model serving consumer
  5. Emit predictions to output topic
Phase 3: 监控（Monitoring） & 可观测性（Observability） (1-2 hours)

Tasks:

  1. Add Prometheus 指标监控（Prometheus） metrics
  2. Create Grafana 可视化（Grafana） dashboard
  3. Implement alerting rules
  4. Set up logging with ELK stack
Success Criteria
  • [ ] Pipeline processes 10K+ records
  • [ ] Data quality score > 95%
  • [ ] Streaming latency < 100ms
  • [ ] All tests passing
  • [ ] 监控（Monitoring） dashboard functional
  • [ ] Documentation complete
