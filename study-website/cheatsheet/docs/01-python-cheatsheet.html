0. Mental model

Python is dynamically typed, interpreted, object-oriented, and has a very rich standard library.
Interview mindset: write clear code first, use built-in data structures, handle edge cases, explain complexity.

Core workflow for coding interviews

1. Restate problem and constraints.

2. Pick data structure: list / dict / set / heap / deque.

3. Write clean function.

4. Test empty / one item / duplicates / large input.

5. Explain time and space complexity.

    arr.append(x)

from collections import deque

def bfs(start):
    q = deque([start])
    seen = {start}
    while q:
        node = q.popleft()
        for nei in graph[node]:
            if nei not in seen:
                seen.add(nei)
                q.append(nei)

DFS

def dfs(node):
    if node in seen:
        return
    seen.add(node)
    for nei in graph[node]:
        dfs(nei)

Binary search

def lower_bound(nums, target):
    left, right = 0, len(nums)
    while left < right:
        mid = (left + right) // 2
        if nums[mid] < target:
            left = mid + 1
        else:
            right = mid
    return left

ADVANCED INTERVIEW DEEP DIVE - EXPANDED COVERAGE

A. What interviewers really test
------------------------------------------------------------
Python interviews test data structure selection, clean state transitions, mutability, complexity, and edge-case discipline. Strong answers are simple, correct, and easy to reason about.

High-signal tools

  • dict/set: average O(1) lookup, dedup, visited states.
  • heapq: top-k, streaming min/max, priority queue.
  • deque: queue, BFS, sliding window.
  • bisect: binary search on sorted arrays.
  • lru_cache: memoized recursion and DP.
  • dataclass: readable structured records.
  • generators: streaming large files without loading all data.

C. Performance deep dive
------------------------------------------------------------

Common traps

  • list.pop(0) is O(n); use deque.popleft().
  • x in list is O(n); use set/dict for repeated lookup.
  • repeated string concatenation can be expensive; collect parts and ''.join(parts).
  • slicing copies; recursive slicing can silently add O(n) per call.
  • sorting is O(n log n); if only top-k is needed, heap can be O(n log k).

Memory model

Assignment binds a name to an object; it does not copy. Lists/dicts/sets are mutable; tuples/strings/ints are immutable. copy.copy is shallow; copy.deepcopy recursively copies nested objects.

D. Production Python for AI/data systems
------------------------------------------------------------

  • Stream large files line by line.
  • Use context managers for files, locks, DB connections, timers.
  • Use logging, metrics, and structured errors instead of print.
  • Use threads for IO-bound work; multiprocessing/Ray/Spark for CPU-bound scale.
  • Understand the GIL: threads do not speed up pure Python CPU loops.
  • Use NumPy/Pandas/PyTorch vectorization for numeric workloads.
from contextlib import contextmanager
import time

@contextmanager
def timer(name):
    start = time.perf_counter()
    try:
        yield
    finally:
        print(f'{name}: {time.perf_counter() - start:.3f}s')

E. Interview Q&A
------------------------------------------------------------
Q: When do you use list, tuple, set, dict?
A: list for ordered mutable sequence, tuple for immutable/hashable records, set for membership/dedup, dict for key-value lookup and counts. For queue semantics, use deque.

Q: Why is dict lookup usually O(1)?
A: It is hash-table based. Average lookup is O(1), but pathological collisions can degrade. Keys must be hashable and equality-consistent.

Q: Shallow vs deep copy?
A: Shallow copy creates a new outer container but references the same nested objects. Deep copy recursively copies nested objects.

Q: How do you debug memory issues?
A: Check large containers/caches, retained references, input growth, and use streaming/generators. Tools: tracemalloc, memory_profiler, objgraph.

Q: How do you write interview-clean Python?
A: State invariant, choose structure, code simply, test edge cases, then explain time/space complexity.
