Section 05

3. nn.Module

import torch.nn as nn

class MLP(nn.Module):
    def __init__(self, input_dim, hidden_dim, num_classes):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(hidden_dim, num_classes),
        )

    def forward(self, x):
        return self.net(x)

Grammar

  • super().__init__() initializes parent nn.Module.
  • layers assigned to self.xxx are registered as parameters.
  • call model(x), not model.forward(x).

4. Loss functions

Multi-class classification

Binary classification

loss = criterion(logits, y.float())

Regression

criterion = nn.MSELoss()

6. Dataset and DataLoader

from torch.utils.data import Dataset, DataLoader

class MyDataset(Dataset):
    def __init__(self, X, y):
        self.X = torch.tensor(X, dtype=torch.float32)
        self.y = torch.tensor(y, dtype=torch.long)

    def __len__(self):
        return len(self.X)

    def __getitem__(self, idx):
        return self.X[idx], self.y[idx]

loader = DataLoader(MyDataset(X, y), batch_size=32, shuffle=True)

Dataset defines how to get one sample. DataLoader handles batching/shuffling.

7. Complete training loop

import torch
import torch.nn as nn

device = "cuda" if torch.cuda.is_available() else "cpu"
model = MLP(input_dim=100, hidden_dim=64, num_classes=3).to(device)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3)

for epoch in range(10):
    model.train()
    total_loss = 0.0

    for xb, yb in train_loader:
        xb = xb.to(device)
        yb = yb.to(device)

        logits = model(xb)
        loss = criterion(logits, yb)

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

        total_loss += loss.item() * xb.size(0)

    avg_loss = total_loss / len(train_loader.dataset)
    print(f"epoch={epoch}, train_loss={avg_loss:.4f}")

8. Evaluation loop

def evaluate(model, loader):
    model.eval()
    correct = 0
    total = 0

    with torch.no_grad():
        for xb, yb in loader:
            xb = xb.to(device)
            yb = yb.to(device)

            logits = model(xb)
            pred = torch.argmax(logits, dim=1)

            correct += (pred == yb).sum().item()
            total += yb.size(0)

    return correct / total

12. Simple text classifier

class TextClassifier(nn.Module):
    def __init__(self, vocab_size, embed_dim, num_classes):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.fc = nn.Linear(embed_dim, num_classes)

    def forward(self, input_ids):
        x = self.embedding(input_ids)          # [B, T, D]
        mask = (input_ids != 0).unsqueeze(-1)  # [B, T, 1]
        x = x * mask
        pooled = x.sum(dim=1) / mask.sum(dim=1).clamp(min=1)
        return self.fc(pooled)

17. Interview talking points

Q: What is autograd?
A: It records tensor operations in a computation graph and computes gradients automatically via backpropagation.

Q: train() vs eval()?
A: They change dropout/batchnorm behavior. eval disables dropout and uses running stats for batchnorm.

Q: Why DataLoader?
A: Batching, shuffling, parallel data loading, and cleaner training loops.

Section 20

ADVANCED INTERVIEW DEEP DIVE - EXPANDED COVERAGE

A. PyTorch mental model
------------------------------------------------------------
PyTorch is eager tensor computation plus autograd. For interviews, connect tensor shapes, autograd graph, module parameters, optimizer state, GPU memory, and distributed training.

B. Shape discipline
------------------------------------------------------------
Most PyTorch bugs are shape bugs.

  • classification logits: [batch, num_classes].
  • CrossEntropy labels: [batch], dtype torch.long.
  • transformer hidden states: often [batch, seq, hidden].
  • attention masks must align with sequence length.
assert logits.shape == (x.size(0), num_classes)
assert y.dtype == torch.long

C. Autograd and memory
------------------------------------------------------------
Autograd records operations for tensors with requires_grad=True. Backward uses saved activations, so training uses much more memory than inference.

Key tools

  • model.eval() changes dropout/batchnorm behavior.
  • torch.no_grad() disables grad recording.
  • torch.inference_mode() is stricter/faster for pure inference.
  • gradient checkpointing trades compute for memory.
  • AMP reduces bandwidth and activation memory.
scaler = torch.cuda.amp.GradScaler()
for xb, yb in loader:
    optimizer.zero_grad(set_to_none=True)
    with torch.cuda.amp.autocast(dtype=torch.float16):
        loss = criterion(model(xb), yb)
    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()

D. Training at scale
------------------------------------------------------------

  • DataParallel: simple, usually slower.
  • DistributedDataParallel: one process per GPU; preferred data parallelism.
  • FSDP/ZeRO: shard parameters, gradients, optimizer states.
  • Tensor parallelism: split matrix multiplies across GPUs.
  • Pipeline parallelism: split layers across GPUs.

Interview answer: DDP replicates the model, each GPU computes gradients for a data shard, then all-reduce averages gradients. It improves throughput but not model capacity. FSDP improves memory capacity by sharding state.

E. Optimization basics
------------------------------------------------------------
AdamW is standard for transformers because it decouples weight decay from adaptive gradient update. Warmup prevents early instability; cosine/linear decay is common. Regularization includes dropout, weight decay, augmentation, early stopping, and label smoothing.

F. Inference performance
------------------------------------------------------------

model.eval()
with torch.inference_mode():
    out = model(x)

torch.compile(model) can reduce Python overhead and fuse operations, but dynamic shapes, unsupported ops, and cold-start compile time can matter.

G. Failure modes
------------------------------------------------------------

  • NaN loss: LR too high, fp16 overflow, bad labels, divide by zero.
  • GPU OOM: batch/sequence too large, retained graph, eval without no_grad.
  • Training slow: dataloader bottleneck, CPU transforms, no pinned memory, tiny batches.
  • Metric mismatch: train/eval mode wrong, label encoding mismatch, data leakage.

H. Interview Q&A
------------------------------------------------------------
Q: Why CrossEntropyLoss expects logits?
A: It combines log-softmax and NLL internally for numerical stability.

Q: What is gradient accumulation?
A: Accumulate gradients across microbatches before optimizer step to simulate larger batch. Usually divide loss by accumulation steps.

Q: DDP vs FSDP?
A: DDP replicates full model and synchronizes gradients. FSDP shards model/optimizer/grad states to fit larger models, at cost of more communication/orchestration.

Q: How do you speed up inference?
A: eval + inference_mode, batching, AMP, compile/export when stable, optimize preprocessing, and use serving systems like Triton/vLLM for production.
