1. Basic text cleaning

import re

def clean_text(text: str) -> str:
    text = text.lower()
    text = re.sub(r"https?://\S+", " URL ", text)
    text = re.sub(r"@\w+", " USER ", text)
    text = re.sub(r"[^a-z0-9\s']", " ", text)
    text = re.sub(r"\s+", " ", text).strip()
    return text

Regex grammar

  • \S+ = one or more non-space chars
  • \s+ = one or more whitespace
  • [^...] = not in this set

6. Sentiment classifier with sklearn

from sklearn.model_selection import train_test_split
from sklearn.pipeline import Pipeline
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import classification_report

texts = [x[0] for x in data]
labels = [x[1] for x in data]

X_train, X_test, y_train, y_test = train_test_split(
    texts, labels, test_size=0.2, random_state=42, stratify=labels
)

model = Pipeline([
    ("tfidf", TfidfVectorizer(ngram_range=(1, 2), min_df=1)),
    ("clf", LogisticRegression(max_iter=1000)),
])

model.fit(X_train, y_train)
pred = model.predict(X_test)
print(classification_report(y_test, pred))

Interview explanation

TF-IDF converts text into sparse features. Logistic Regression learns feature weights for each label.

12. Fine-tuning transformer classifier

from transformers import AutoTokenizer, AutoModelForSequenceClassification
from transformers import Trainer, TrainingArguments
from datasets import Dataset

model_name = "distilbert-base-uncased"
tokenizer = AutoTokenizer.from_pretrained(model_name)

def tokenize_batch(batch):
    return tokenizer(batch["text"], truncation=True, padding="max_length", max_length=128)

train_ds = Dataset.from_dict({"text": train_texts, "label": train_labels})
val_ds = Dataset.from_dict({"text": val_texts, "label": val_labels})
train_ds = train_ds.map(tokenize_batch, batched=True)
val_ds = val_ds.map(tokenize_batch, batched=True)

model = AutoModelForSequenceClassification.from_pretrained(model_name, num_labels=3)

args = TrainingArguments(
    output_dir="./out",
    eval_strategy="epoch",
    save_strategy="epoch",
    learning_rate=2e-5,
    per_device_train_batch_size=16,
    per_device_eval_batch_size=16,
    num_train_epochs=3,
)

trainer = Trainer(model=model, args=args, train_dataset=train_ds, eval_dataset=val_ds)
trainer.train()

13. Semantic search example

from sentence_transformers import SentenceTransformer
import numpy as np

docs = [
    "How to reset my password",
    "Shipping policy and delivery time",
    "Refund and return instructions",
]
encoder = SentenceTransformer("sentence-transformers/all-MiniLM-L6-v2")
doc_emb = encoder.encode(docs, normalize_embeddings=True)

def search(query, k=3):
    q_emb = encoder.encode([query], normalize_embeddings=True)
    scores = (q_emb @ doc_emb.T)[0]
    idx = np.argsort(scores)[::-1][:k]
    return [(docs[i], float(scores[i])) for i in idx]

print(search("I want my money back"))

15. Interview Q&A

Q: Bag-of-words vs embeddings?
A: Bag-of-words/TF-IDF are sparse lexical features. Embeddings are dense semantic vectors.

Q: Why n-grams?
A: Single words lose phrase meaning. Bigrams capture phrases like "not good".

Q: What is OOV?
A: Out-of-vocabulary tokens unseen during training. Subword tokenizers reduce OOV.

Q: Why transformers?
A: Self-attention models contextual relationships between tokens and supports parallel training.

Section 18

ADVANCED INTERVIEW DEEP DIVE - EXPANDED COVERAGE

A. NLP system mental model
------------------------------------------------------------
NLP systems map raw text into normalized tokens, features, embeddings, predictions, and explanations. Modern interviews expect both classic NLP and transformer-based workflows.

B. Text preprocessing decisions
------------------------------------------------------------
Common steps: Unicode normalization, language detection, sentence splitting, tokenization, optional lowercasing, PII redaction, and metadata extraction. Do not blindly remove punctuation/stopwords; enterprise queries often include ticket IDs, error codes, filenames, and product names.

C. Tokenization deep dive
------------------------------------------------------------

  • whitespace/regex: simple, brittle.
  • WordPiece/BPE/SentencePiece: handles rare words via subwords.
  • character-level: robust but longer sequences.

Transformer pitfalls

  • length is tokens, not words.
  • truncation can drop evidence.
  • tokenizer must match model training.
  • chunking policy strongly affects RAG/search quality.

D. Embeddings and similarity
------------------------------------------------------------

Embedding design questions

  • symmetric or query/document-specific encoders?
  • multilingual support?
  • domain vocabulary support?
  • exact ID/number preservation?
  • normalized vectors for cosine similarity?
# Conceptual semantic similarity
q = normalize(encoder([query])[0])
d = normalize(encoder(docs))
scores = d @ q

E. Classification, NER, ranking
------------------------------------------------------------

Task framing

  • classification: intent, toxicity, category.
  • NER: people, orgs, products, dates, tickets.
  • ranking: query-document relevance.
  • generation: grounded answer, usually with retrieval.

Reranking pattern

1. retrieve top 100/1000 cheaply.

2. cross-encoder or LLM reranks a small candidate set.

3. apply freshness, permissions, diversity, and business rules.

F. Evaluation
------------------------------------------------------------

  • classification: precision, recall, F1, ROC-AUC, PR-AUC.
  • ranking: MRR, nDCG, Recall@K.
  • extraction: entity-level precision/recall/F1.
  • generation/RAG: groundedness, answer relevance, citation correctness.

G. Failure modes
------------------------------------------------------------

  • domain shift from web text to enterprise tickets.
  • token truncation silently removes evidence.
  • dense embeddings miss exact IDs/SKUs.
  • data leakage through duplicate docs/users/time.
  • multilingual mismatch.
  • metric looks good overall but fails critical slices.

H. Interview Q&A
------------------------------------------------------------
Q: BM25 vs dense embeddings?
A: BM25 is strong for exact terms, IDs, rare tokens, and explainability. Dense embeddings capture paraphrase and semantic similarity. Production search often uses hybrid retrieval.

Q: Why cross-encoder reranker?
A: A bi-encoder is fast because document vectors are precomputed. A cross-encoder jointly reads query and document, usually more accurate but too expensive for the full corpus.

Q: How do you chunk documents for NLP/RAG?
A: Preserve semantic/page/section boundaries, use overlap only when helpful, keep metadata/ACLs, and tune size by retrieval recall and context budget.

Q: What is data leakage in NLP?
A: Same/near-duplicate documents, users, or time-correlated examples appearing in both train and test inflate metrics without real generalization.
