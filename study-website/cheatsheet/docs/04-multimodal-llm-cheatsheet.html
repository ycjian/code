Section 04

2. Prompting vision models

Screenshot prompt

Analyze the screenshot.
Return:
1. Main task/question
2. Important constraints
3. Step-by-step solution
4. Edge cases
If text is unclear, say what is uncertain.

Chart prompt

Read the chart carefully.
Extract axes, units, trend, outliers, and conclusion.
Do not invent exact values if the chart does not show them clearly.

Document extraction prompt

Extract the fields as JSON.
Use null if missing.
Preserve original wording for legal/financial fields.

Base64 image pattern

import base64
from openai import OpenAI

client = OpenAI()

def encode_image(path):
    with open(path, "rb") as f:
        return base64.b64encode(f.read()).decode("utf-8")

img_b64 = encode_image("screenshot.png")

response = client.responses.create(
    model="YOUR_MULTIMODAL_MODEL",
    input=[{
        "role": "user",
        "content": [
            {"type": "input_text", "text": "Extract the coding question."},
            {"type": "input_image", "image_url": f"data:image/png;base64,{img_b64}"},
        ],
    }],
)

Interview note

Most APIs accept image URL or base64 data URL. Keep image size reasonable. Use clear output schema.

Section 07

5. CLIP-style embeddings

Use CLIP-like models for image search by text, image dedup, recommendation, multimodal retrieval.

Concept

text encoder maps text -> vector
image encoder maps image -> vector
similarity(text_vector, image_vector) finds matching images

Example

from sentence_transformers import SentenceTransformer, util
from PIL import Image

model = SentenceTransformer("clip-ViT-B-32")
images = [Image.open("dog.jpg"), Image.open("car.jpg")]
image_emb = model.encode(images, convert_to_tensor=True, normalize_embeddings=True)
text_emb = model.encode(["a photo of a dog"], convert_to_tensor=True, normalize_embeddings=True)
scores = util.cos_sim(text_emb, image_emb)
print(scores)

8. Video understanding pattern

Do not send every frame. Sample frames.

import cv2

def sample_frames(video_path, every_n_seconds=2):
    cap = cv2.VideoCapture(video_path)
    fps = cap.get(cv2.CAP_PROP_FPS)
    step = int(fps * every_n_seconds)
    frames = []
    i = 0
    while True:
        ok, frame = cap.read()
        if not ok:
            break
        if i % step == 0:
            frames.append(frame)
        i += 1
    cap.release()
    return frames

Pipeline

video -> sampled frames -> frame captions -> temporal summary -> answer

Section 15

13. Interview answer pattern

Question: Design a document understanding system.

Answer

I would split this into ingestion and query-time flows. At ingestion, I parse PDFs into pages, extract text/OCR/tables, preserve metadata like page number and bounding boxes, chunk the text, create embeddings, and store them in a vector database. At query time, I embed the query, retrieve top-k chunks/pages, optionally rerank, and pass the grounded context to a multimodal or text LLM. For evaluation, I measure retrieval recall, answer correctness, and citation accuracy.

Section 16

ADVANCED INTERVIEW DEEP DIVE - EXPANDED COVERAGE

A. Multimodal LLM mental model
------------------------------------------------------------
A multimodal system aligns text, image, video, audio, documents, layout, and metadata. The engineering problem is model quality plus payload size, latency, routing, privacy, and evaluation.

B. Architecture patterns
------------------------------------------------------------

1. Dual encoder: encode image/text separately into shared space; good for retrieval.

2. Cross encoder: jointly attends over modalities; better quality, slower.

3. Vision-language adapter: visual encoder feeds tokens/features into LLM.

4. Tool-based multimodal agent: LLM calls OCR, detector, retriever, video sampler.

5. Cascade: cheap router first, expensive VLM only when needed.

C. Representation choices
------------------------------------------------------------
Images: resize/crop policy, aspect ratio, perceptual hash, OCR. Video: keyframes, transcript, scene detection, timestamps. Documents: text, layout, tables, images, page regions. Always store metadata that lets the answer cite the original page/frame/region.

D. Multimodal retrieval
------------------------------------------------------------

Use hybrid retrieval

  • BM25/OCR for exact text.
  • dense multimodal vectors for semantic/visual match.
  • metadata filters for tenant, app, modality, time, permissions.
  • reranker to fuse query, asset, thumbnails, captions, and behavior.

Example: "red logo slide from last week's deck" needs lexical terms, visual color/logo similarity, slide/document metadata, and recency filters.

E. Serving and latency
------------------------------------------------------------
Large media is expensive. Use presigned uploads, edge/client preprocessing, thumbnails/keyframes, content-hash caching, async jobs for long video/doc tasks, and route only necessary derived artifacts to GPU services.

F. Evaluation
------------------------------------------------------------

  • retrieval: Recall@K, nDCG, MRR.
  • captioning/VQA: human preference, accuracy, hallucination rate.
  • RAG: groundedness and citation correctness.
  • system: p95 latency, payload bytes, GPU cost per request.

G. Failure modes
------------------------------------------------------------

  • modality dominance: metadata overwhelms visual evidence.
  • OCR misses or hallucinates text.
  • frame sampling misses the key moment.
  • dense embeddings fail exact UI text or numbers.
  • raw media goes to cloud despite privacy constraints.
  • expensive VLM used for every trivial request.

H. Interview Q&A
------------------------------------------------------------
Q: How do CLIP-like models enable text-to-image search?
A: They train image and text encoders into a shared vector space so related text/image pairs are close. At query time, encode text and ANN-search image vectors.

Q: Why keep lexical search in multimodal systems?
A: Exact filenames, OCR text, IDs, and policy metadata are hard for dense embeddings. Hybrid retrieval improves recall and explainability.

Q: How reduce raw-media transfer?
A: Edge-first preprocessing, presigned upload, content hashes, thumbnails/keyframes, cached embeddings/OCR, and only send necessary derived artifacts.

Q: How serve long video understanding?
A: Async workflow: transcript/keyframes -> chunk index with timestamps -> retrieve relevant segments -> run VLM on narrowed evidence -> cite timestamps.
