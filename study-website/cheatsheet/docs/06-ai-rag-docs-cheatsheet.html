Section 02

0. RAG mental model

RAG = Retrieval-Augmented Generation.

Instead of asking the LLM from memory

1. Retrieve relevant documents/chunks.

2. Put them into prompt as context.

3. Ask LLM to answer grounded in context.

4. Return answer with citations.

中文

RAG 的目标是让 LLM 基于你的文档回答,降低 hallucination,并支持更新的私有知识。

    doc = fitz.open(path)
    chunks = []
    for page_idx, page in enumerate(doc):
        text = page.get_text()
        chunks.append({
            "text": text,
            "metadata": {"source": path, "page": page_idx + 1}
        })
    return chunks
Section 05

3. Chunking

Naive chunking

def chunk_text(text, chunk_size=800, overlap=100):
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunks.append(text[start:end])
        start = end - overlap
    return chunks

Better chunking

  • split by sections/headings
  • preserve paragraphs
  • keep tables together
  • keep code blocks together
  • add overlap
  • attach metadata

Chunk size guidance

Short FAQ/support: 200-500 tokens
General docs: 500-1000 tokens
Legal/technical docs: 800-1500 tokens
Tables: keep as whole logical unit

Tradeoff

Small chunks: high precision, may lose context
Large chunks: more context, lower retrieval precision

7. Simple FAISS retrieval

import faiss
import numpy as np

vectors = np.array(vectors).astype("float32")
dim = vectors.shape[1]
index = faiss.IndexFlatIP(dim)  # inner product
index.add(vectors)

def retrieve(query_vector, k=5):
    q = np.array([query_vector]).astype("float32")
    scores, ids = index.search(q, k)
    return list(zip(ids[0], scores[0]))

If embeddings are normalized: inner product == cosine similarity.

Section 13

11. Reranker example

from sentence_transformers import CrossEncoder

reranker = CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2")

pairs = [(query, doc["text"]) for doc in retrieved_docs]
scores = reranker.predict(pairs)
ranked = sorted(zip(retrieved_docs, scores), key=lambda x: x[1], reverse=True)
top_docs = [doc for doc, score in ranked[:5]]

Why rerank

Vector search is fast but approximate. Cross-encoder compares query+doc together and improves precision.

Section 15

13. Context packing

Bad: dump top 20 chunks blindly.

Good

  • remove duplicates
  • group by source/page
  • sort by relevance
  • include metadata
  • fit token budget
def build_context(docs, max_chars=8000):
    parts = []
    total = 0
    for d in docs:
        header = f"[source={d['source']}, page={d.get('page')}]"
        text = header + "\n" + d["text"]
        if total + len(text) > max_chars:
            break
        parts.append(text)
        total += len(text)
    return "\n\n---\n\n".join(parts)

20. Interview answer pattern

Question: Design RAG over company documents.

Answer

I would build an ingestion pipeline that parses documents, chunks them by semantic structure, attaches metadata such as source, page, section, version, and ACL, then embeds chunks and stores them in a vector database. At query time, I would apply permission filters, retrieve using hybrid search, rerank candidates, construct compact context with citations, and ask the LLM to answer only from retrieved context. Section 23

ADVANCED INTERVIEW DEEP DIVE - EXPANDED COVERAGE

A. RAG mental model
------------------------------------------------------------
RAG grounds generation in external knowledge. The hardest parts are ingestion quality, permissions, retrieval recall, context assembly, citation correctness, and evaluation.

B. End-to-end pipeline
------------------------------------------------------------

1. Ingest connectors.

2. Parse text/layout/tables/images/OCR.

3. Chunk by semantic/page/section boundaries.

4. Compute embeddings and lexical fields.

5. Store chunks with ACLs, tenant, source, version, timestamp.

6. Hybrid retrieve.

7. Rerank/dedup/diversify.

8. Build prompt with citations.

9. Generate answer.

10. Evaluate and log feedback.

C. Chunking deep dive
------------------------------------------------------------
Small chunks improve precision but lose context. Large chunks preserve context but waste tokens and reduce precision. Overlap helps continuity but creates duplicates. Structure-aware chunking is best for policies, PDFs, code, and tables.

D. Retrieval strategies
------------------------------------------------------------
Hybrid retrieval combines BM25, vector search, metadata filters, query rewriting, and reranking. Use RRF when scores are not calibrated; use learned fusion when labels exist.

E. Context assembly
------------------------------------------------------------
Context is scarce. Select diverse chunks, remove near-duplicates, preserve hierarchy, include citations, and keep instructions separate from retrieved content. Compress evidence only after retrieval, not before recall is solved.

F. Permissions and privacy
------------------------------------------------------------
Apply ACL and tenant filters during retrieval, before generation. Never let unauthorized chunks enter the prompt. Cache ACL-aware results carefully and version documents to avoid stale leaks.

G. Evaluation
------------------------------------------------------------

  • retrieval recall@K.
  • answer faithfulness.
  • citation precision.
  • refusal correctness when evidence is missing.
  • latency/cost per answer.
  • human/LLM judge agreement.

H. Failure modes
------------------------------------------------------------
Stale index, ACL leak through cache, semantically similar wrong policy, prompt injection in documents, correct retrieval but ungrounded generation, and model ignoring citations.

I. Interview Q&A
------------------------------------------------------------
Q: How improve hallucinating RAG?
A: Separate retrieval failure from generation failure. Improve chunking/recall/reranking if evidence missing; improve prompt constraints, citation validation, and refusal logic if evidence exists.

Q: Why hybrid search?
A: Enterprise queries contain exact names, IDs, acronyms, and policy terms. Dense handles paraphrase; BM25 handles exact/rare terms.

Q: How handle document updates?
A: Version documents, incremental index, tombstone old chunks, track embedding model version, and exclude stale/deleted chunks at query time.

Q: How evaluate citations?
A: Compare answer claims to cited chunks with human labels or calibrated LLM-as-judge, and track citation precision/recall on hard examples.
