Module 01: Foundations of 机器学习基础设施（ML Infrastructure）

Duration: 40 hours Difficulty: Beginner Prerequisites: Basic Python programming, command-line familiarity

Module Overview

Welcome to the first module of your AI 基础设施工程师（AI Infrastructure Engineer）ing journey! This module establishes the foundational concepts, tools, and practices you'll need throughout your career in ML infrastructure.

You'll learn what ML infrastructure is, why it matters, and how to set up a professional development environment. By the end of this module, you'll deploy your first ML model using a basic serving framework.

学习产出（Learning Outcomes）

By the end of this module, you will be able to:

  1. Understand the ML Lifecycle - Explain the stages of ML development from training to production
  2. Set Up Development Environment - Configure Python, Docker 容器（Docker）, Git, and cloud tools for ML infrastructure work
  3. Deploy First ML Model - Use TorchServe or TensorFlow 深度学习框架（TensorFlow） Serving to deploy a pre-trained model
  4. Understand the Role of Infrastructure - Articulate how infrastructure enables ML systems
  5. Navigate ML Frameworks - Work with PyTorch 深度学习框架（PyTorch） and TensorFlow 深度学习框架（TensorFlow） at a basic level
  6. Apply MLOps 机器学习运维（MLOps） Fundamentals - Understand key MLOps 机器学习运维（MLOps） concepts and their importance

Topics Covered

1. Introduction to 机器学习基础设施（ML Infrastructure） and MLOps 机器学习运维（MLOps）
  • What is ML infrastructure?
  • The difference between ML engineering and ML infrastructure engineering
  • MLOps 机器学习运维（MLOps）: Definition, importance, and maturity levels
  • Industry trends and career opportunities
2. The ML Lifecycle
  • 训练（Training） phase: data, model development, experimentation
  • Serving phase: deployment, inference, monitoring
  • Feedback loop: performance tracking, retraining
  • The role of infrastructure at each stage
3. ML Frameworks Overview
  • PyTorch 深度学习框架（PyTorch）: dynamic computation graphs, research-friendly
  • TensorFlow 深度学习框架（TensorFlow）: static graphs, production-optimized
  • Hugging Face Transformers: pre-trained models and ecosystems
  • When to use which framework
4. Development Environment Setup
  • Python 3.9+ with virtual environments
  • Git for version control
  • Docker 容器（Docker） basics and installation
  • Cloud platform setup (AWS/GCP 云（GCP）/Azure 云（Azure）)
  • VS Code or PyCharm configuration for ML work
5. Cloud Platforms Introduction
  • Infrastructure as a Service (IaaS) vs Platform as a Service (PaaS)
  • Major cloud providers: AWS 云（AWS）, GCP 云（GCP）, Azure 云（Azure）
  • Core services for ML: compute, storage, networking
  • Free tiers and cost management basics
6. 模型服务（Model Serving） Fundamentals
  • What is model serving?
  • Serving frameworks: TorchServe, TensorFlow 深度学习框架（TensorFlow） Serving, FastAPI 服务框架（FastAPI）
  • REST APIs for model inference
  • Synchronous vs asynchronous serving

Common Questions

Q: Do I need a GPU 加速（GPU） for this module? A: No, a CPU is sufficient for this module. We'll work with small models and pre-trained weights. GPU 加速（GPU） will be important in later modules.

Q: Which cloud platform should I choose? A: Any of AWS 云（AWS）, GCP 云（GCP）, or Azure 云（Azure） will work. If unsure, start with GCP 云（GCP） (generous free tier) or AWS 云（AWS） (most widely used in industry). You can try multiple platforms.

Q: How long should each lesson take? A: The time estimates are guidelines. Expect 3-8 hours per lesson depending on your background. Take your time and understand concepts deeply.

Q: What if I get stuck on an exercise? A: Check the solutions folder, review the lesson materials, search official documentation, and post questions in GitHub Discussions. Learning to debug and find solutions is part of the skill development.

Q: Can I skip the environment setup if I already have Python/Docker 容器（Docker） installed? A: Review the setup lesson anyway to ensure your environment matches the curriculum requirements. Missing dependencies will cause issues in later modules.

Tips for Success

  1. Hands-on practice is essential - Don't just read, type the code and run examples
  2. Document your learning - Keep notes on commands, errors, and solutions
  3. Build your portfolio - Save all exercise code in a personal GitHub repository
  4. Join the community - Connect with other learners for support and knowledge sharing
  5. Take breaks - ML infrastructure concepts can be dense. Step away and return with fresh eyes.
  6. Ask questions - No question is too basic. Understanding fundamentals is crucial.
