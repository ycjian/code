Project 2 Deep Dive: Multimodal Embeddings

Project 2 深入：多模态 Embedding

Why Embeddings Matter

为什么 Embedding 重要

Creative search is often intent-based rather than keyword-based. A user may search for "cinematic winter street scene," upload an example image, ask for "similar but warmer," or browse related assets from an existing design. This requires models that map text, image, video frames, audio, 3D previews, and document thumbnails into comparable representations.

创意搜索通常是 intent-based，而不是 keyword-based。用户可能搜索 “cinematic winter street scene”，上传参考图，要求 “similar but warmer”，或者从已有设计中浏览相关 asset。这需要模型把 text、image、video frame、audio、3D preview 和 document thumbnail 映射到可比较的 representation。

A natural architecture is a dual-encoder or multi-encoder system inspired by CLIP-style contrastive learning. Text and media encoders produce vectors in a shared space. Similar text-media pairs are pulled together, while unrelated pairs are pushed apart.

一个自然的架构是 dual-encoder 或 multi-encoder 系统，类似 CLIP-style contrastive learning。文本和媒体 encoder 产出同一空间中的向量。相关的 text-media pair 被拉近，不相关的 pair 被推远。

Model Design

模型设计

Representative encoder design:

代表性的 encoder 设计：

  • Text encoder: query text, asset captions, generated metadata, OCR text, transcript snippets, product taxonomy.
  • 文本 encoder：query text、asset caption、生成 metadata、OCR text、transcript snippet、产品 taxonomy。
  • Image encoder: still images, thumbnails, frame crops, masks, visual style features.
  • 图像 encoder：静态图、thumbnail、frame crop、mask、视觉风格 feature。
  • Video encoder: sampled keyframes, scene-level aggregation, temporal metadata, transcript alignment.
  • 视频 encoder：采样 keyframe、scene-level aggregation、temporal metadata、transcript alignment。
  • Document encoder: text chunks, page thumbnails, layout regions, table or heading structure.
  • 文档 encoder：text chunk、page thumbnail、layout region、table 或 heading structure。
  • 3D encoder: rendered views, mesh metadata, material tags, preview thumbnails.
  • 3D encoder：rendered view、mesh metadata、material tag、preview thumbnail。

The practical production design is often not a single giant model. It is a family of encoders with compatibility contracts. Some encoders may share the same embedding dimension and distance metric. Some may require projection heads or calibration. Some may be used only for candidate generation, while cross-encoders or rankers refine the top candidates.

实际生产设计通常不是单一巨型模型，而是一组有兼容性契约的 encoder。有些 encoder 共享 embedding dimension 和 distance metric，有些需要 projection head 或 calibration。有些只用于 candidate generation，而 cross-encoder 或 ranker 用于 refine top candidates。

Contrastive Objective

对比学习目标

For a batch of paired text and media examples, the model learns to assign higher similarity to matching pairs than to non-matching pairs. CLIP used a contrastive objective over image-text pairs. ALIGN showed that noisy large-scale image-text pairs can also work when scale and filtering are strong. SigLIP later showed that a sigmoid loss can be an effective alternative to softmax contrastive loss for language-image pretraining.

对于一批 paired text/media example，模型学习让匹配 pair 的相似度高于不匹配 pair。CLIP 使用 image-text pair 上的 contrastive objective。ALIGN 表明在规模和过滤足够强时，noisy large-scale image-text pair 也可以有效。SigLIP 进一步表明 sigmoid loss 可以作为 language-image pretraining 中 softmax contrastive loss 的有效替代。

Interview explanation:

面试表达：

For multimodal search, I would usually start with a dual-encoder architecture because it supports fast retrieval over large catalogs. The model learns a shared space for queries and assets. The tradeoff is that dual encoders are fast but less expressive than cross-encoders, so I use them for recall and then rerank a smaller candidate set.

对于多模态搜索，我通常会从 dual-encoder 架构开始，因为它支持大规模 catalog 上的快速 retrieval。模型学习 query 和 asset 的共享空间。tradeoff 是 dual encoder 很快，但表达力不如 cross-encoder，所以我用它做 recall，再对较小 candidate set 做 rerank。

  • Human labels: curated text descriptions, tags, editorial metadata.
  • 人工标签：curated text description、tag、editorial metadata。
  • Product metadata: asset titles, captions, creator tags, folder/project context.
  • 产品 metadata：asset title、caption、creator tag、folder/project context。

  • Weak labels: generated captions, OCR text, transcripts, automatic tags.

The danger is noisy positives. A click does not always mean the result was relevant. A tag may describe only one aspect of an image. A generated caption may miss style or composition. In production, the data pipeline needs weighting, filtering, deduplication, and auditability.

风险是 noisy positive。点击不一定代表结果相关，tag 可能只描述图片的一个方面，生成 caption 可能遗漏风格或构图。生产中，数据 pipeline 需要 weighting、filtering、deduplication 和 auditability。

Negative Pairs And Hard Negative Mining

负样本与 Hard Negative Mining

Negatives matter as much as positives. In-batch negatives are cheap, but many are too easy. For creative search, hard negatives are assets that look close but violate intent: same color but wrong object, same object but wrong style, same semantic category but wrong license, same transcript but wrong shot, visually similar but unsafe or policy-invalid.

负样本和正样本同样重要。in-batch negative 成本低，但很多太简单。对于创意搜索，hard negative 是那些看起来接近但违反 intent 的 asset：颜色相同但物体不对，物体相同但风格不对，语义类别相同但 license 不对，transcript 相同但镜头不对，视觉相似但 unsafe 或 policy-invalid。

Hard negatives can be mined from:

hard negative 可以从这些地方挖掘：

  • Current retrieval mistakes: high-ranked but skipped or downvoted assets.
  • 当前 retrieval mistake：排名高但被跳过或 downvote 的 asset。
  • Query-result logs: impressions without clicks, or clicks followed by quick abandonment.
  • query-result log：曝光但未点击，或点击后快速放弃。
  • A/B test regressions: queries where new model hurts relevance or diversity.
  • A/B test regression：新模型伤害 relevance 或 diversity 的 query。
  • Near neighbors in vector space that fail metadata or human relevance checks.
  • vector space 中相近但未通过 metadata 或人工相关性检查的 neighbor。
  • Cross-modal confusions: image similar to query text but wrong style, layout, or domain.
  • 跨模态混淆：图像与 query text 相似但风格、layout 或 domain 不对。

Interview explanation:

面试表达：

I would not treat hard-negative mining as a one-time offline step. I would connect it to product logs and evaluation failures. If the system keeps returning black photos for "black wedding invitation template," those false positives become hard negatives so the next model learns the difference between color similarity and product intent.

我不会把 hard-negative mining 看成一次性离线步骤，而是把它接到产品日志和评估失败上。如果系统总是对 “black wedding invitation template” 返回黑色照片，这些 false positive 就应该进入 hard negative，让下一版模型学会区分颜色相似和产品 intent。

Why Retrieval And Ranking Are Separate

为什么 Retrieval 和 Ranking 要分开

Retrieval optimizes recall under a tight latency budget. Ranking optimizes precision, ordering, diversity, business rules, and product utility. A dual encoder can retrieve thousands of candidates quickly. A ranker can spend more compute on a smaller candidate set.

Prompting, RAG, Fine-Tuning, And Model Training

Prompting、RAG、Fine-Tuning 与模型训练

For product-facing LLM workflows, the modeling decision is often not "train a bigger model." It is choosing the right adaptation mechanism:

对于面向产品的 LLM workflow，modeling 决策通常不是“训练更大的模型”，而是选择合适的适配机制：

  • Prompting: fastest iteration, but fragile and harder to guarantee.
  • Prompting：迭代最快，但更脆弱，难以保证一致性。
  • RAG: best when the answer depends on product assets, documents, user context, or enterprise knowledge.
  • RAG：当答案依赖 product asset、document、user context 或 enterprise knowledge 时最合适。

陷阱：“LLM-as-Judge 告诉我们模型是否好。”

Better:

更好：

LLM-as-Judge is one semantic signal. I combine it with deterministic checks, human-labeled evals, offline replay, canary metrics, and product telemetry.

LLM-as-Judge 是一个 semantic signal。我会结合 deterministic check、human-labeled eval、offline replay、canary metrics 和 product telemetry。

Trap: "We just fine-tune for product behavior."

陷阱：“我们直接 fine-tune 来获得产品行为。”

Better:

更好：

I first diagnose whether the problem is knowledge, behavior, latency, cost, or style. RAG, prompting, adapters, distillation, and fine-tuning solve different problems.

我会先诊断问题是 knowledge、behavior、latency、cost 还是 style。RAG、prompting、adapter、distillation 和 fine-tuning 解决的是不同问题。
